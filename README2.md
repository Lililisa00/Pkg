Описание реализации лабораторной работы
Язык программирования: Python 3.x Внешняя библиотека: Pillow (PIL - Python Imaging Library) v.10.x. Графический интерфейс: Tkinter.
Способ получения данных: Приложение использует библиотеку Pillow для чтения заголовков файлов. Чтение метаданных производится без полной декомпрессии растровых данных (пикселей), что обеспечивает высокое быстродействие при обработке больших массивов данных (до 100 000 файлов).
Пояснение выводимых характеристик:
	Размер изображения (в пикселях):
	Что это: Физическое количество точек по горизонтали и вертикали.
	Откуда берется: Свойства img.width и img.height объекта Image.
  Разрешение (DPI - Dots Per Inch):
	Что это: Плотность пикселей, указывающая, сколько точек приходится на один дюйм при печати. Важно для полиграфии, но не влияет на отображение на мониторе.
	Откуда берется: Словарь img.info['dpi']. Обычно считывается из заголовков EXIF (для JPG) или pHYs (для PNG).
	Глубина цвета (Color Depth):
	Что это: Количество бит, используемых для кодирования цвета одного пикселя. Определяет максимальное количество цветов (например, 24 бита = 16.7 млн цветов).
	Откуда берется: Свойство img.mode. Программа преобразует режимы библиотеки Pillow в биты:
	1 -> 1 бит (черно-белое)
	L или P -> 8 бит (256 цветов/градаций)
	RGB -> 24 бита (TrueColor)
	RGBA -> 32 бита (TrueColor + Альфа-канал)
	Сжатие:
	Что это: Алгоритм, использованный для уменьшения размера файла.
	Откуда берется: Словарь img.info['compression'].
	JPG использует сжатие с потерями (JPEG DCT).
	PNG использует сжатие без потерь (Deflate).
	GIF использует LZW.
	BMP часто хранится без сжатия или использует RLE (Run-Length Encoding).
